# 📝 Task Groups & Admin Tasks API — Гайд по процессу

## 📋 Описание

Этот гайд описывает процесс создания и управления группами заданий и отдельными заданиями через API платформы DrSarha. Используйте коллекцию `postman-task-groups-admin-tasks-api.json` для тестирования.

---

## 🚦 Процесс создания: Сначала группа, потом задания

### 1. **Создайте группу заданий**

```http
POST /task-groups/
Content-Type: application/json

{
  "name": "Группа для новичков",
  "description": "Задания для новых пользователей",
  "startDate": "2024-06-01",
  "endDate": "2024-12-31",
  "rewardStars": 100,
  "rewardExp": 200,
  "level": 1
}
```

- **Ответ:**

```json
{
  "_id": "6655f1234567890123456789",
  ...
}
```

- Скопируйте `groupId` из ответа для следующих шагов.

---

### 2. **Добавьте задания в группу**

```http
POST /task-groups/{groupId}/tasks
Content-Type: application/json

{
  "title": "Создать первый пин",
  "description": "Пользователь должен создать свой первый пин",
  "actionType": "create_pin",
  "targetAmount": 1,
  "rewardStars": 10,
  "rewardExp": 20
}
```

- **Ответ:**

```json
{
  "_id": "6655f1234567890123456790",
  ...
}
```

---

### 3. **Получите все группы (админ)**

```http
GET /task-groups/admin/all
```

---

### 4. **Обновите или удалите группу**

```http
PUT /task-groups/{groupId}
DELETE /task-groups/{groupId}
```

---

### 5. **Работа с отдельными заданиями (не в группе)**

- Для создания, обновления, удаления и просмотра заданий вне групп используйте эндпоинты `/admin/tasks/` (требуется JWT токен администратора):

```http
POST /admin/tasks/
GET /admin/tasks/
PUT /admin/tasks/{taskId}
DELETE /admin/tasks/{taskId}
```

---

## 🗂️ Примеры для Postman

- В коллекции есть переменные `{{groupId}}` и `{{taskId}}` — подставляйте их из ответов на создание.
- Для админских методов `/admin/tasks/` используйте заголовок:
  ```
  Authorization: Bearer {{admin_jwt}}
  ```

---

## 📝 Примечания

- **Сначала всегда создавайте группу, потом добавляйте задания!**
- Для каждой группы можно создать любое количество заданий.
- Для отдельных (негрупповых) заданий используйте только `/admin/tasks/`.
- Все методы требуют корректные данные (см. коллекцию для примеров).
